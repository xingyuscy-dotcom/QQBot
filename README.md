# QQbot_v2

基于 NapCat + OneBot v11 + FastAPI 的 QQ 机器人。

当前目标是：登录一个 QQ 账号，把 NapCat 作为消息入口，接入 DeepSeek API，实现群聊和私聊的自动问答、会话隔离、后台配置、长期记忆、命令管理、日志、备份和统计。

## 功能

- NapCat 反向 WebSocket 接入 QQ 消息。
- 群聊和私聊分会话隔离，私聊也作为独立会话。
- DeepSeek OpenAI-compatible API 调用。
- 网页后台配置 API、人设、会话、记忆、命令库、备份和日志。
- 支持全局人设和单独会话人设。
- 支持管理员长期记忆和会话学习记忆。
- 支持会话级学习记忆影响强度和上下文条数配置。
- 支持群聊触发模式：仅 @、前缀、全部消息。
- 支持会话启用 / 停用、回复冷却、回复概率、每小时回复上限。
- 支持本地日志、备份、恢复、统计表格和折线图。
- 支持管理员命令和群聊真正 @ 指定 QQ。
- 支持本地知识库和独立热点库。
- 支持知识库自动触发、强制检索、模型总结和敏感度配置。
- 支持独立热点库，普通聊天和 `/热点` 命令都可以优先检索热点。
- 支持微博历史热搜补录，程序运行时每天 0 点自动补录上一天热搜。
- 支持后台命令库展开 / 收起、新增、删除和保存。
- 支持会话调试，能查看最近上下文、最终提示词、知识库 / 热点库命中和测试回复。

## 环境

- Windows 10/11
- PowerShell
- Python 3.11 或更高版本
- NapCat

Python 依赖在首次启动时会自动安装到项目内 `.venv`。

## 启动

进入项目目录：

```powershell
cd 你的仓库目录
```

启动：

```powershell
.\start.ps1
```

也可以双击仓库里的快捷启动脚本：

```text
start-qqbot-v2.bat
```

启动成功后会自动打开后台网页，并在命令行输出：

```text
Local web: http://127.0.0.1:6185/
NapCat reverse websocket: ws://127.0.0.1:6199/onebot/ws
```

后台地址：

```text
http://127.0.0.1:6185/
```

如果不想自动打开浏览器，可以运行：

```powershell
.\start.ps1 -NoOpenBrowser
```

## NapCat 配置

在 NapCat 中添加反向 WebSocket，地址填：

```text
ws://127.0.0.1:6199/onebot/ws
```

如果机器人和 NapCat 不在同一台机器，把 `127.0.0.1` 改成运行 QQbot_v2 的机器 IP。

## 后台配置

打开后台：

```text
http://127.0.0.1:6185/
```

至少需要配置：

- `Base URL`：默认 `https://api.deepseek.com`
- `API Key`：你的 DeepSeek API Key
- `Model`：`deepseek-v4-flash` 或 `deepseek-v4-pro`
- `管理员 QQ`：允许执行管理命令的 QQ 号，多个用英文逗号分隔

保存后，API Key 会写入本地 `config.local.json`，后台不会明文回显。

知识库相关配置：

- `知识库`：总开关，关闭后普通聊天不自动检索知识库和热点库。
- `知识库敏感度`：控制普通聊天触发检索的积极程度。
- `知识库最大条数`：控制每次交给模型参考的本地条目数量。
- `知识库强制前缀`：例如 `查知识库,知识库`，用户消息以这些前缀开头时会强制检索知识库。

每个会话可以单独配置：

- 会话人格设定
- 管理员长期记忆
- 学习批量
- 学习记忆影响
- 上下文条数，默认 `8`，填 `0` 表示关闭上下文读取
- 回复模式、回复冷却、回复概率、每小时回复上限

后台还提供：

- 命令库管理：命令列表默认收起，可以展开 / 收起、新增命令、删除命令，保存后生效。
- 会话调试：查看最近消息、提示词、知识库 / 热点库命中预览，并生成一次不会发送到 QQ 的测试回复。
- 备份恢复：创建备份、查看备份内容、用备份恢复当前数据，恢复前会自动创建安全备份。
- 信息统计：按 `1 天`、`7 天`、`30 天` 查看每个会话的接收量、输出量、模型请求数和 token 用量。

## 测试

群聊收发测试：

```text
@机器人 测试
```

如果返回下面内容，说明 OneBot 通道正常：

```text
QQbot_v2 已收到群消息，OneBot 通道正常。
```

管理员群聊 @ 测试：

```text
@机器人 /at 目标QQ号 测试一下
```

例如：

```text
@机器人 /at 123456789 测试一下
```

私聊测试：

```text
测试
```

命令帮助：

```text
/帮助
```

知识库总结测试：

```text
/知识库总结 BLG 最近有什么热点
```

热点库测试：

```text
/热点 BLG
```

普通聊天自动热点测试：

```text
@机器人 BLG 最近有啥热点
```

如果热点库有匹配数据，会优先参考热点库；如果没命中，会继续按正常聊天回复，不会固定回复“没查到”。

上下文测试：

```text
我最近在纠结要不要换电脑
旧电脑有点卡
@机器人 那你觉得我该怎么办
```

机器人应该只回答最后一句，但能理解前面两句提供的上下文。

## 常用命令

- `/帮助`：查看当前可用命令。
- `/状态`：查看当前会话状态，管理员专用。
- `/启用`：启用当前会话机器人，管理员专用。
- `/停用`：停用当前会话机器人，管理员专用。
- `/模式 查看`：查看当前群聊回复模式，管理员专用。
- `/模式 @机器人`：群聊仅 @ 机器人时回复，管理员专用。
- `/模式 前缀 /bot`：群聊使用指定前缀触发，管理员专用。
- `/模式 全部消息`：所有消息都触发回复，管理员专用。
- `/人设 查看`：查看当前会话人设，管理员专用。
- `/人设 设置 内容`：设置当前会话人设，管理员专用。
- `/人设 清空`：清空当前会话人设，管理员专用。
- `/记忆 查看`：查看当前会话管理员长期记忆，管理员专用。
- `/记忆 添加 内容`：添加当前会话管理员长期记忆，管理员专用。
- `/记忆 删除 编号`：删除指定管理员长期记忆，管理员专用。
- `/记忆 清空`：清空当前会话管理员长期记忆，管理员专用。
- `/学习 查看`：查看当前会话学习记忆，管理员专用。
- `/学习 更新`：立即更新当前会话学习记忆，管理员专用。
- `/学习 开启`：开启当前会话学习，管理员专用。
- `/学习 关闭`：关闭当前会话学习，管理员专用。
- `/学习 批量 数量`：设置当前会话学习批量，`0` 表示使用全局批量，管理员专用。
- `/学习 权重 0.4`：设置学习记忆影响强度，范围 `0` 到 `1`，管理员专用。
- `/学习 清空缓存`：清空当前会话待学习消息数，管理员专用。
- `/知识库 内容`：快速检索本地知识库并直接返回命中条目，管理员专用。
- `/知识库总结 内容`：检索本地知识库后调用大模型整理回答，普通用户可用。
- `/热点`：查看本地热点榜单，普通用户可用。
- `/热点 平台或关键词`：按平台或关键词检索热点库，普通用户可用。
- `/备份`：创建本地备份，管理员专用。
- `/备份列表`：查看最近备份，管理员专用。
- `/健康检查`：查看运行健康状态，管理员专用。
- `/at QQ号 内容`：群聊真正 @ 指定 QQ，管理员专用。

## 知识库和热点库

后台知识库区域支持：

- 手动更新知识库和当前热点。
- 搜索测试知识库。
- 查看知识库和热点库统计。
- 补录历史热搜，开始日期取后台输入框，结束日期默认取当前系统日期。
- 显示脚本运行进度、预计剩余时间和脚本输出。

知识库和热点库的区别：

- `/知识库 内容`：管理员专用，快速检索本地知识库并直接返回命中条目。
- `/知识库总结 内容`：普通用户可用，检索本地知识库后调用大模型整理回答。
- `/热点`：普通用户可用，查看或搜索本地热点库。
- 普通聊天里出现“热点、热搜、最近、今天、什么情况”等意图时，会优先检索热点库；没命中时走正常聊天。

当前热点更新会尝试抓取微博、百度、知乎、B站、GitHub 等公开热榜，部分来源可能因为反爬或网络状态跳过。

程序运行时会启动一个轻量后台任务，每天 0 点自动补录上一天微博历史热搜，成功补录日期会记录在 `hot.last_daily_archive_date`。历史热搜来自第三方归档，只作为热点线索，不等于事实结论。

也可以用脚本手动补录：

```powershell
.\.venv\Scripts\python.exe .\scripts\update_hot_history.py --from 2025-01-01 --to 2026-07-08
```

只补录某一天：

```powershell
.\.venv\Scripts\python.exe .\scripts\update_hot_history.py --date 2026-07-07
```

## 部署到新机器

1. 安装 Python 3.11 或更高版本。
2. 安装并登录 NapCat。
3. 克隆仓库，进入仓库目录。
4. 运行 `.\start.ps1`，或双击 `start-qqbot-v2.bat`。
5. 打开 `http://127.0.0.1:6185/`。
6. 填入 DeepSeek API Key、模型、管理员 QQ。
7. 在 NapCat 中配置反向 WebSocket：`ws://127.0.0.1:6199/onebot/ws`。
8. 在 QQ 群或私聊发送测试消息。
